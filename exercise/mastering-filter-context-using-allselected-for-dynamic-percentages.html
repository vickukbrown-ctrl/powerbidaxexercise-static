Mastering Filter Context: Using ALLSELECTED for Dynamic Percentages

Difficulty: Intermediate

üìö Theory & Explanation

The ALLSELECTED function is a powerful DAX modifier used within CALCULATE to remove all filter contexts from the current query, *except* for filters that have been applied manually by the user (e.g., through slicers or report-level filters). This allows you to calculate a 'total' that is relative to the user's current selections, rather than the entire dataset or the specific visual's filter context, making it ideal for 'percentage of total' calculations that adapt to user interaction.

üß© Example Use Case

As a Power BI analyst, you'll frequently use `CALCULATE` with `ALLSELECTED` and `DIVIDE` to compute percentage of subtotal metrics that dynamically respond to user selections. For example, to show each product's sales contribution *within the currently selected category or region*, you'd `CALCULATE` the relevant subtotal using `ALLSELECTED` (to remove filters from the current visual but keep outer slicer filters), and then `DIVIDE` the individual product's sales by that subtotal to get an interactive, context-aware percentage. This allows your users to slice data and immediately see relative performance within their specific focus area, rather than always against the grand total.

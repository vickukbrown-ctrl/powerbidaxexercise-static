Advanced Filtering with DAX: Using FILTER for Context Modification

üßÆ Function: FILTER, CALCULATE, COUNTROWS

Difficulty: Intermediate

üìö Theory & Explanation

The FILTER function is an iterator that evaluates a Boolean expression for each row of a table or an expression that returns a table. It returns a table containing only those rows for which the expression is TRUE. This virtual table can then be passed to other functions, such as aggregation functions or CALCULATE, to perform complex calculations based on a derived set of data. When used as a filter argument within CALCULATE, FILTER can modify the filter context, allowing measures to aggregate data not just on direct column values but also on conditions involving related tables or complex logic.

üß© Example Use Case

As an analyst, you'll frequently need to count rows that meet specific, dynamic conditions across related tables. For instance, to count how many `Customers` have placed at least one high-value order (e.g., over $500), you would combine these functions: `CALCULATE(COUNTROWS(Customers), FILTER(Sales, Sales[OrderAmount] > 500))`. This uses `CALCULATE` to modify the filter context, applying a `FILTER` on the `Sales` table to ensure `COUNTROWS` only considers `Customers` associated with those specific, high-value transactions.

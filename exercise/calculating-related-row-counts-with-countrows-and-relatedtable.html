Calculating Related Row Counts with COUNTROWS and RELATEDTABLE

üßÆ Function: COUNTROWS, RELATEDTABLE

Difficulty: Intermediate

üìö Theory & Explanation

COUNTROWS is a DAX function that counts the number of rows in a specified table. When used in a row context (like within a calculated column), it counts rows based on the current row's context. RELATEDTABLE is a DAX function that retrieves a table related to the current row, filtered by the current row's context. Combining COUNTROWS(RELATEDTABLE()) allows you to efficiently count all related rows from a 'many' side table for each row in the 'one' side of a relationship, directly within a calculated column.

üß© Example Use Case

When you need to count how many related items exist for each row in your current table‚Äîfor example, calculating the number of orders a specific customer has placed, or the number of products within a particular category‚Äîyou'll combine `COUNTROWS` with `RELATEDTABLE`. `RELATEDTABLE` first dynamically retrieves the table of *only* the related child items for the current parent row, and then `COUNTROWS` efficiently counts the rows within that filtered table, providing a powerful way to summarize one-to-many relationships directly in your model.
